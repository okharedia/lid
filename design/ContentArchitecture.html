How your content fits together

The app has one primary question source of truth, one selected-language translation catalog, and derived learner surfaces such as glossary highlights and question notes. The useful distinction: official German question/answer text is protected source content; translations, notes, and glossary explanations are learner-facing layers.

310questions: 300 general + 10 Berlin
1,240answers, four per question

Rules Of Ownership

            
These rules keep edits predictable and stop derived content from becoming accidental source content.

          

          

            

              
 German source text

              
Questions and answers live in the source-of-truth JSON. Do not patch generated app state when source text is wrong.

            

            

              
 Translation text

              
Translations belong in the locale catalog. UI copy should avoid assuming one target language.

            

            

              
 Glossary text

              
data/glossary.json is generated from question metadata glossary refs. Change extraction rules or context overrides, then rebuild.

Current important caveat

              

                The generated glossary currently stores translated labels and learner context from data/i18n/en.json.
                If the product supports multiple translation languages, the durable architecture should either store translation
                keys in the glossary or generate locale-specific glossary files.
