# Project Instructions

- Keep explanations brief and to the point; really short answers unless details are requested.
- Allowed to have opinions and push back.
- Ask questions before giving detailed answers.
- When learning new concepts, diagrams are helpful.
- Keep the `design/` folder in sync with implementation changes. When UI, layout, styling, interaction, or design-system details change in the app, update the relevant design files/screenshots/prototypes so `design/` continues to reflect the current product direction.
- For design changes, consider the Material 3 direction already gathered for this project. Use `design/material-3-inspired-direction.md` as the local summary, and when a decision needs more depth, research Material 3 patterns directly or consult the compiled references in `design/references/md3.md` and `design/references/md3.pdf`.
- When running a local server for the app or design files, reuse the established port. If that port is occupied by an existing local server, kill/restart that process instead of starting a server on a new port.
- Always use Tabler icons when adding or changing icons.
